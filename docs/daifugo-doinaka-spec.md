# 大富豪 ド田舎ルール仕様書（ドラフト v0.1）

最終更新: 2026-07-19  
対象: 大富豪（2人対戦）

## 1. 目的

ド田舎ルールを、2人対戦向けの「防衛失敗ペナルティ」ルールとして定義する。  
本仕様は実装前の合意用ドキュメントであり、コード実装は別タスクで行う。

## 2. 用語

- 防衛者: 前ゲームの勝者。
- 防衛戦: 防衛者が参加する次ゲーム。
- 防衛成功: 防衛戦で防衛者が再び勝者になること。
- ド田舎成立: 防衛戦で防衛者が敗北すること。
- ド田舎ペナルティ: ド田舎成立時に次ゲーム開始時へ持ち越される不利効果。

## 3. 適用範囲

- 対象モード:
  - PvP（2P LOCAL）: 適用対象。
  - CPU対戦（LOCAL CPU）: 適用対象。
  - ルーム対戦: 非適用（クラシック固定）。
- ルールトグル名: ド田舎ルール。
- 初期値: OFF。
- プリセット保存: ON/OFF状態を保存・復元する。

## 4. 基本挙動

### 4.1 判定タイミング

- 各ゲーム終了時に、防衛戦中かどうかを判定する。
- 防衛戦中で、かつ防衛者が敗北した場合にド田舎成立とする。

### 4.2 ペナルティ内容

- ド田舎成立時、次ゲーム開始時に一度だけ以下を適用する。
  - ド田舎成立者の手札から最強札を1枚選び、相手へ渡す。
  - 相手の手札から最弱札を1枚選び、ド田舎成立者へ渡す。
- 交換後は通常のゲーム進行へ入る。
- ペナルティ適用後、保留フラグは必ず消去する。

### 4.3 最強/最弱の判定

- 判定はそのゲーム開始時点の強さ基準を使う。
- 強さ反転が有効な場合（革命、11バックなど）:
  - 反転状態を考慮して最強/最弱を決定する。
- 同強さが複数ある場合:
  - スート順のタイブレークを既存実装準拠で使用する。

## 5. 他ルールとの優先順位

ゲーム開始時の処理順は以下とする。

1. 配札
2. 勝者敗者でカード交換（強⇔弱）
3. ド田舎ペナルティ交換
4. 先手決定（前回勝者先攻 / 3♦先攻 / ランダム）

補足:
- ド田舎ルールと勝者敗者でカード交換は併用可能。
- 両方ONの場合は、上記順序で連続適用する。

## 6. 状態管理

実装時に保持する論理状態の要件を以下に定義する。

- previousWinner: 前ゲーム勝者（既存）。
- defendingChampion: 防衛者のプレイヤー番号（0/1 または null）。
- doinakaPenaltyPendingPlayer: 次ゲーム開始時にペナルティ適用対象となるプレイヤー番号（0/1 または null）。

状態遷移:
- ゲーム終了時:
  - 防衛者が存在し、かつ敗北したら doinakaPenaltyPendingPlayer に防衛者を設定。
  - 防衛成功時は doinakaPenaltyPendingPlayer を null にする。
- 新ゲーム開始時:
  - doinakaPenaltyPendingPlayer が設定されていればペナルティを実行し、null に戻す。
  - defendingChampion には previousWinner を設定して次の防衛戦へ進める。

## 7. UI/表示要件

- ルールメニューに「ド田舎ルール」チェック項目を表示。
- ド田舎成立時のメッセージ例:
  - 「ド田舎成立: 次ゲームでカード交換ペナルティ」
- ペナルティ適用時のメッセージ例:
  - 「ド田舎ペナルティ: 最強札と最弱札を交換しました」

## 8. 受け入れ基準

- ON時のみ、ド田舎成立判定と次ゲーム開始時ペナルティが発動する。
- OFF時は発動しない。
- プリセット保存/読み込みでド田舎ON/OFFが保持される。
- ルーム対戦では常に非適用。
- 既存ルール（革命、11バック、カード交換、先攻決定）と同時有効時も処理順が再現される。

## 9. 非目的

- 本仕様書作成時点では、コード実装・UI演出追加・ネットワーク同期拡張は行わない。

## 10. 実装メモ（次タスク向け）

- 判定関数を分離し、ゲーム終了時と開始時で副作用を局所化する。
- 交換処理は既存のカード交換ロジックを再利用し、重複実装を避ける。
- ルーム同期へ含めるかは、ルームモードがクラシック固定のため不要。
