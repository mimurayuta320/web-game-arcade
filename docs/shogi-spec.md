# 将棋 仕様書（実装準拠）

## 1. 文書目的
この文書は、Neon Board Arcade 内の将棋機能について、現行実装に基づく仕様を定義する。

## 2. 対象ファイル
- UI: index.html
- ゲームロジック: src/scripts/shogi.js
- 画面スタイル: src/styles/main.css
- 画面起動・文言: src/main.js

## 3. ゲームモード
- ローカル2人: 人間同士で1端末対戦
- CPU対戦: 人間 vs CPU
- ルーム対戦: WebSocketルーム経由のオンライン対戦
- カオス: 2人将棋ベースで開始時ハンデを適用

## 4. 盤面と人数
- 全モード: 9x9

## 5. 駒構成
### 5.1 2人（通常）
- 標準将棋配置（9x9）

### 5.2 4人（カオス）
未実装

## 6. 手番と開始手
### 6.1 2人
- 開始手: 先攻/後攻/ランダム

### 6.2 4人カオス
未実装

## 7. 基本ルール
- 駒移動: 将棋準拠
- 成り: 任意成りと必須成りあり
- 持ち駒: 取得駒を手駒化して打てる
- 打ち制約:
  - 二歩禁止
  - 打ち歩詰め禁止
  - 行き所のない駒打ち禁止（歩/香/桂）
- 王手判定: あり

## 8. 勝敗判定
### 8.1 2人モード
- 王が取られたら敗北
- 合法手なし（詰み/ステイル）判定あり

### 8.2 4人カオス
未実装

## 9. カオス設定
カオス時に以下を設定できる。
- 先手ハンデ
- 後手ハンデ

## 10. プレイヤー別設定UI
カオス設定パネル内で、先手/後手ごとにハンデを設定する。

## 11. ハンデ仕様
選択肢:
- なし
- 飛車落ち
- 角落ち
- 二枚落ち（飛角）
- 四枚落ち（飛角香香）

適用タイミング:
- 新規対局開始時（START）
- 待機盤面（standby）表示時

## 12. 地雷仕様
未実装

## 13. 王吸収モード
未実装

## 14. CPU仕様
- 難易度: easy / normal / hard / pro
- 候補手の上限、ランダム率、読み深さを難易度ごとに変更

## 15. ルーム対戦仕様
- 6桁ルーム番号で参加
- ホスト/ゲスト/観戦ロールを持つ
- 設定同期対象:
  - 盤面・手駒・手番（対局状態）
  - 指し手（move / drop）

## 16. 表示仕様
- 2人将棋表示（先手/後手）
- カオス選択時のみ、先手/後手ハンデ設定パネルを表示

## 17. 既知の制約
