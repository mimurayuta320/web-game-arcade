# 将棋 仕様書（実装準拠）

## 1. 文書目的
この文書は、Neon Board Arcade 内の将棋機能について、現行実装に基づく仕様を定義する。

## 2. 対象ファイル
- UI: index.html
- ゲームロジック: src/scripts/shogi.js
- 画面スタイル: src/styles/main.css
- 画面起動・文言: src/main.js

## 3. ゲームモード
- ローカル2人: 人間同士で1端末対戦
- CPU対戦: 人間 vs CPU
- ルーム対戦: WebSocketルーム経由のオンライン対戦
- カオス: 追加ルール付き将棋

## 4. 盤面と人数
- 通常/カオス2人: 9x9
- カオス4人: 13x13
- 4人時の陣営順: 北 -> 東 -> 南 -> 西

## 5. 駒構成
### 5.1 2人（通常）
- 標準将棋配置（9x9）

### 5.2 4人（カオス）
各陣営が以下を持つ。
- 王 x1
- 金 x1
- 銀 x1
- 角 x1
- 飛 x1
- 香 x1
- 桂 x1
- 歩 x5

## 6. 手番と開始手
### 6.1 2人
- 開始手: 先攻/後攻/ランダム

### 6.2 4人カオス
- 開始プレイヤー: 1/2/3/4/ランダム
- 手番巡回: 北 -> 東 -> 南 -> 西

## 7. 基本ルール
- 駒移動: 将棋準拠
- 成り: 任意成りと必須成りあり
- 持ち駒: 取得駒を手駒化して打てる
- 打ち制約:
  - 二歩禁止
  - 打ち歩詰め禁止
  - 行き所のない駒打ち禁止（歩/香/桂）
- 王手判定: あり

## 8. 勝敗判定
### 8.1 2人モード
- 王が取られたら敗北
- 合法手なし（詰み/ステイル）判定あり

### 8.2 4人カオス
- 現行実装では2人用の終局判定を適用していない（専用終局判定は未整備）

## 9. カオス設定
カオス時に以下を設定できる。
- 対局人数（2/4）
- 王吸収（ON/OFF）
- プレイヤー別ハンデ
- プレイヤー別地雷数

## 10. プレイヤー別設定UI
カオス設定パネル内で、プレイヤーごとに設定する。
- 一人目の設定
  - ハンデ
  - 地雷数
- 二人目の設定
  - ハンデ
  - 地雷数
- 三人目の設定（4人時のみ表示）
  - ハンデ
  - 地雷数
- 四人目の設定（4人時のみ表示）
  - ハンデ
  - 地雷数

## 11. ハンデ仕様
選択肢:
- なし
- 飛車落ち
- 角落ち
- 二枚落ち（飛角）
- 四枚落ち（飛角香香）

4人時は陣営ごとの初期配置に対して個別適用する。

## 12. 地雷仕様
### 12.1 配置
- 設定数に応じて地雷駒をランダム配置
- 2人時: 先手側/後手側で個別設定
- 4人時: 北/東/南/西で個別設定

### 12.2 挙動
- 地雷駒を取ると、取った側の王以外の駒は消滅（移動駒が盤上から消える）
- 地雷駒自体は捕獲扱いになり、対応駒が手駒に加算される

## 13. 王吸収モード
### 13.1 発動条件
- 王吸収ON
- 王が味方駒を取る
- 同一タイプ吸収済みでない

### 13.2 効果
- 取った駒タイプの能力を王に追加
- 異なる駒タイプは累積可能
- 同一タイプは重複不可（既に持っていれば吸収不可）

### 13.3 移動拡張ルール
- 王は通常8方向1マス移動
- 吸収した能力の方向ベクトルを加算し、該当方向の移動可能距離を延長
- 向き依存駒（歩/桂/香/銀/成り）も駒の向きを考慮して能力化

## 14. CPU仕様
- 難易度: easy / normal / hard / pro
- 候補手の上限、ランダム率、読み深さを難易度ごとに変更

## 15. ルーム対戦仕様
- 6桁ルーム番号で参加
- ホスト/ゲスト/観戦ロールを持つ
- 設定同期対象:
  - 先後（開始側）
  - ハンデ設定
  - 地雷設定
  - 王吸収設定
  - 対局人数（2/4）

## 16. 表示仕様
- 4人時は駒向き（north/south/east/west）を描画
- 4人時は駒表示色を陣営別テーマで描画
- 手番表示は陣営名（北/東/南/西）を使用

## 17. 既知の制約
- 4人カオスの終局判定は専用仕様が未完了
- 手駒管理は2色ベース実装を拡張して運用しており、4陣営完全独立手駒の仕様化は今後検討余地あり
