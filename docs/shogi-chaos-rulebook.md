# 将棋カオスモード ルールブック（ドラフト v0.1）

## 1. 目的

通常将棋の戦略性を残しつつ、毎局異なる展開になる「イベント型ルール」を導入する。

- ベースは既存の将棋ルール（成り、持ち駒、二歩、打ち歩詰め禁止、王手/詰み）
- カオス要素はゲームを壊さない範囲で段階的に発生
- 初見でも遊べるよう、各イベントは短い文でUI表示する

## 2. 基本仕様

### 2.1 盤と勝敗

- 盤面: 9x9
- 勝利条件: 相手玉を詰ませる（または相手が合法手なし）
- 敗北条件: 自玉が詰み、または反則扱いの操作

### 2.2 通常ルールの継承

以下は通常将棋と同様に適用する。

- 駒の基本移動
- 成り（任意成り/必須成り）
- 持ち駒の打ち
- 二歩禁止
- 打ち歩詰め禁止
- 自玉放置手の禁止

## 3. カオスゲージとイベント発生

### 3.1 カオスゲージ

- 初期値: 0
- 毎手番終了時に +1
- 駒を取った手番は追加で +1
- 王手をかけた手番は追加で +1

### 3.2 イベント発生判定

- ゲージが `5` 以上で判定対象
- 手番開始時に確率判定し、成功でイベント発生
- 発生確率: `min(20 + (ゲージ - 5) * 10, 70)%`
- イベント発生後、ゲージは `0` に戻る

## 4. イベント一覧（実装候補）

1局で使うイベントは以下からランダム抽選する。

### 4.1 盤面系

1. 重力
- 次の1手のみ、移動後の駒が可能な限り前方へ1マス滑る
- 玉と金は対象外

2. 霧
- 次の2手、相手の合法手ハイライトを表示しない

3. 地割れ
- ランダム1マスを「封鎖マス」にする
- 封鎖マスには移動/打ち不可
- 3手経過で解除

### 4.2 持ち駒系

1. 強制ドロップ
- 発動側は次の自手番で、可能なら持ち駒を1枚打たなければならない

2. 逆輸入
- 相手の持ち駒ランダム1枚を自分の持ち駒に移す
- 相手が持ち駒ゼロなら不発

3. 暴発成り
- 次の1手、成れる駒は必ず成る（任意成りを強制化）

### 4.3 手番系

1. 疾風
- 発動側は次の手番で持ち時間を半減（ターン制限を導入している場合）

2. 時間凍結
- 相手の次手番のみ持ち時間消費を停止（同上）

3. 反転命令
- 次の1手だけ「駒を取る手」を禁止
- 取る手しか合法手がない場合は通常ルール優先で例外的に許可

## 5. イベント適用の優先順位

同時に複数の効果が競合する場合、次の順で処理する。

1. 反則防止（自玉放置、二歩、打ち歩詰め禁止）
2. 勝敗確定（詰み判定）
3. 強制系イベント（強制ドロップ、暴発成り）
4. 盤面系イベント（地割れ、重力、霧）
5. 表示演出

## 6. 反則・救済ルール

- イベントにより合法手が0になる場合:
  - そのイベントは不発として扱う
  - 不発でも「イベント発生演出」は残してよい（ログ上は不発と表示）
- UI誤操作対策:
  - イベント適用中の禁止操作はクリック時に理由を表示
- 再現性確保:
  - 乱数シードを対局開始時に固定し、対局ログに保持する

## 7. ルーム対戦向け同期仕様（推奨）

- ホストがシード値を生成して全員へ配布
- イベント抽選は「手数 + シード」で決定
- クライアント側で同じ結果を再生できること
- 送信データは「手」「イベントID」「適用対象」の最小限にする

## 8. UI表示ルール

- 手番開始時にトースト表示: `カオス発動: 地割れ（3手）`
- 盤面効果はアイコンと残り手数を表示
- 対局ログに履歴を残す
  - 例: `12手目: 先手イベント「逆輸入」発動（角を獲得）`

## 9. バランス調整パラメータ（初期値）

- ゲージ増加量: 通常 `+1` / 取得 `+1` / 王手 `+1`
- 発生確率下限: 20%
- 発生確率上限: 70%
- 1対局あたり目標発生回数: 3〜6回
- 「盤面破壊系（地割れ）」の出現重み: 低め（重み1）
- 「情報系（霧）」の出現重み: 中（重み2）
- 「持ち駒系」の出現重み: 中〜高（重み3）

## 10. 段階導入プラン

1. Phase 1（最小）
- 霧
- 暴発成り
- 逆輸入

2. Phase 2（中規模）
- 地割れ
- 強制ドロップ

3. Phase 3（上級）
- 重力
- 反転命令
