# 五目並べ 仕様書（v1.0）

## 1. 目的

本仕様書は、Neon Board Arcade に実装されている五目並べ（Neon Gomoku）の機能要件、画面要件、対戦モード仕様、同期仕様、受け入れ条件を定義する。

- 対象: ローカル対戦、CPU対戦、ルーム対戦（PvP）
- 対象外: 棋譜保存、観戦専用UIの拡張、AI難易度段階化

## 2. ゲーム概要

- 盤面サイズ: 15x15
- 先手: 黒（BLACK）
- 後手: 白（WHITE）
- 勝利条件: 自石を縦・横・斜めのいずれかで5連続以上にする
- 引き分け条件: 全マスが埋まり、勝者がいない

## 3. 対戦モード

### 3.1 ローカル2人対戦（2P LOCAL）

- 1台端末で黒白を交互に操作する。
- 手番プレイヤーのみ着手可能。

### 3.2 CPU対戦（VS CPU）

- 人間プレイヤー: 黒
- CPU: 白
- CPU手番時は操作不可とし、思考中メッセージを表示する。

### 3.3 ルーム対戦（PvP）

- ルーム接続時にモードはルーム対戦へ遷移する。
- ホストは黒、ゲストは白を担当する。
- ゲーム開始操作はホストのみ可能。
- ルーム画面に ROOM / ROLE を表示する。

## 4. 画面仕様

### 4.1 HUD

- 表示項目
- TURN: 現在の手番（BLACK / WHITE）
- MODE: VS CPU / 2P LOCAL（ルーム対戦中は変更不可）
- ボタン: GAME START / MENU

### 4.2 盤面表示

- 15x15 グリッドを正方形で表示する。
- 盤面は画面サイズに応じて可変（レスポンシブ）表示。
- 石の見た目は黒石・白石で視覚的に区別可能であること。

### 4.3 オーバーレイ表示

- 待機時: GAME STARTで開始
- 終局時: BLACK WIN / WHITE WIN / DRAW
- ルームロック時: 待機メッセージを表示

### 4.4 メッセージ表示

- ローカル2P: 黒の手番です / 白の手番です
- CPU対戦: あなたの手番です / CPUの手番です / CPUが考えています...
- ルーム対戦: あなたの手番です / 相手の手番です

## 5. 操作仕様

### 5.1 基本操作

- 空マスのクリックで着手する。
- 着手済みマスへの操作は無効。
- 対局終了後は盤面操作を無効化する。

### 5.2 GAME START

- ローカル / CPU: 新規対局を開始する。
- ルーム対戦: ホストのみ有効。開始時に相手へ新規対局イベントを同期する。

### 5.3 MENU

- 確認ダイアログで承認後、前画面へ戻る。
- ルーム対戦中はロビーへ戻る。

## 6. 判定ロジック

### 6.1 5連判定

- 着手点を起点に以下4方向を判定する。
- 縦（上下）
- 横（左右）
- 斜め（左上-右下）
- 斜め（右上-左下）
- 双方向の連結数合計が5以上で勝利とする。

### 6.2 引き分け判定

- 盤面に空マスが存在しない場合に DRAW とする。

## 7. CPU思考仕様

### 7.1 思考タイミング

- CPU手番開始時に一定遅延後（約260ms）で着手する。

### 7.2 着手優先順位

1. CPUが即勝利できる手を選ぶ。
2. 相手の即勝利手をブロックする。
3. 上記がない場合、以下の評価で最良手を選ぶ。
- 盤中央への近さ
- 隣接8方向の自石・敵石の密度
- 完全同点時の微小ランダム

### 7.3 候補制約

- 近傍に石が存在する空マスを優先候補とする。
- 候補がない初期局面では全空マスを対象とする。

## 8. ルーム同期仕様

### 8.1 同期イベント

- select-game: ロビーでゲーム選択を同期
- new-game: 新規対局開始を同期
- move: 着手座標（row, col）を同期
- snapshot: 盤面・手番・終局状態などのスナップショットを同期

### 8.2 スナップショット項目

- board
- currentPlayer
- gameOver
- roomLocked
- roomLockMessage
- message
- overlay

### 8.3 ロック制御

- 対戦相手不在時は roomLocked を有効化し、盤面操作を無効にする。
- 再接続・再開時はロック解除後に対局可能とする。

## 9. 状態遷移

- 待機（Standby）
- 対局中（Playing）
- 終局（Finished）
- ルーム待機ロック（Room Locked）

遷移条件:

- GAME START: Standby -> Playing
- 勝敗/引き分け確定: Playing -> Finished
- ルーム相手離脱: Playing -> Room Locked
- メニュー戻りまたは再開始: Finished/Room Locked -> Standby

## 10. 非機能要件

- 操作応答: 人間着手はクリック後即時に反映する。
- CPU応答: 体感待ち時間を演出しつつ、1秒未満で応答する。
- 表示互換: PC/スマートフォンの主要表示幅で盤面が収まること。
- 同期一貫性: ルーム両端末で盤面・手番・終局表示が一致すること。

## 11. 受け入れ条件（Acceptance Criteria）

1. VS CPU を選択して開始すると、白番でCPUが自動着手する。
2. 2P LOCAL を選択すると、CPUは動作せず黒白を人間操作できる。
3. ルーム対戦ではホストのみ GAME START が有効になる。
4. ルーム対戦中に双方で同一の着手結果が再現される。
5. 5連成立時に正しい勝者表示とオーバーレイ表示が出る。
6. 全マス埋まり時に DRAW 表示になる。
7. 終局後は盤面操作が無効化される。
