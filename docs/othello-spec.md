# オセロ仕様書

最終更新: 2026-07-18
対象実装: `src/scripts/game.js` / `index.html` / `src/styles/main.css` / `src/main.js`

## 1. 概要

本仕様は、本プロジェクトのオセロ実装（Neon Othello）の現行挙動を定義する。
通常ルールのオセロに加え、CPU対戦、CPU同士対戦、ルーム対戦、カオス拡張ルールを提供する。

## 2. 対応モード

### 2.1 標準モード
- `cpu`
  - 人間1人 vs CPU1人。
  - 先後を `先攻（あなた）/後攻（あなた）/ランダム` から選択可能。
- `cpuvscpu`
  - CPU同士の自動対戦。
- `local`
  - ローカル2人対戦。

### 2.2 カオスモード
- `chaos`
- 基本はオセロ盤面だが、以下の拡張ルールを有効化する。
  - 上書き配置
  - 固定石（反転不可）
  - 駒破壊スキル
  - 2回行動スキル
  - 最終集計時のランダム1列カウント除外（ON/OFF）

### 2.3 ルームモード
- ゲーム内部モードは `room`。
- ルーム作成/参加後、ホストが `local` または `chaos` 相当を選択して同期対戦する。
- 非ホストはモード・各種設定変更が制限される。

## 3. 盤面・初期配置

- 盤面サイズ: 8x8
- 初期配置:
  - `d4 = 白`, `e4 = 黒`, `d5 = 黒`, `e5 = 白`
- 手番開始: 黒

## 4. 基本ルール

### 4.1 合法手判定
- 8方向（縦横斜め）を探索し、相手石を1つ以上はさんで自石で閉じる配置を合法手とする。
- 通常モードでは空マスのみ配置可能。
- カオスモードでは条件付きで相手石への上書き配置も合法手に含む（詳細は 8.1）。

### 4.2 反転
- 着手位置を自色にした後、はさんだ相手石を自色へ反転。
- 固定石に指定された石は反転対象から除外される。

### 4.3 パス
- 次手番側に合法手がない場合はパスし、手番を維持して続行する。
- 両者とも合法手がない場合は終局。

## 5. タイマー

- 1手の持ち時間: 30秒
- ルームモードではローカルタイマーを進行させない。
- タイムアウト時:
  - 合法手がある場合: 合法手からランダムに1手を自動実行。
  - 合法手がない場合: 時間切れ敗北（相手勝利）。

## 6. 終局・勝敗

- 終局条件:
  - 両者合法手なし
  - 盤面が全て埋まった
  - タイムアウト敗北
  - 合意引き分け成立（ローカル/ルーム）
- 通常集計は黒石数と白石数で判定。
- カオスで「ランダム1列カウント除外」がONの場合、終局時にランダムな1行または1列を集計対象外として勝敗判定する。

## 7. CPU仕様

### 7.1 難易度
- `easy`（かんたん）
- `normal`（ふつう）
- `hard`（つよい）
- `expert`（さいきょう）
- `pro`（プロ）

### 7.2 思考特性
- 難易度ごとに以下パラメータを変更。
  - 思考時間
  - ランダム着手率
  - 角/辺/X/Cマス重み
  - 機動力重み
  - 局面評価重み
  - 探索深さ・分岐上限
  - 終盤探索強化（上位難易度）
- `pro` はオープニングブックを使用し、深い探索を優先。

### 7.3 学習要素
- 人間vsCPU時、着手傾向とCPUバイアスをローカル保存し、CPU評価へ反映する。
- 学習データキー: `neon-othello-learning-v1`（ユーザースコープ保存対応）。
- 一部高難易度では学習バイアスを無効化し、決定論的探索を優先する。

## 8. カオス拡張仕様

### 8.1 上書き配置
- 自手番で相手石があるマスにも配置可能（残回数がある場合）。
- 上書き時は残回数を1消費。
- 固定石には上書き不可。
- 初期値は片側3回（対象に応じて個別設定可能）。

### 8.2 ハンデ対象
- 対象指定: `なし / 黒 / 白 / あなた / 相手 / 両者`
- 対象側に対して、固定石チャージ・上書き回数・駒破壊回数を適用可能。

### 8.3 固定石（immutable1）
- ハンデ `固定石 +1（反転不可）` を持つ側は、条件を満たすと1回だけ自石を固定化できる。
- 固定可能条件:
  - 内部マス（外周不可）
  - 現在その側の石である
  - 未固定である
- 固定後は反転対象外。

### 8.4 駒破壊スキル
- 発動可能回数はサイド別に設定可能。
- 固定石は破壊対象外。
- 発動パターン:
  - 角犠牲パターン
    - 自角石1個を犠牲にして、相手石3個をランダム破壊。
  - 非角パターン
    - 自石2個を犠牲にして、相手石1個を破壊。
- 発動後は手番を進行。

### 8.5 2回行動
- ストックを消費して、1手後に同一プレイヤーへ手番を継続できる。
- ストック獲得条件:
  - 角を2回連続で失うごとに1ストック獲得。
- 発動予約中に着手すると1ストック消費。

### 8.6 角取得時ボーナス
- 対局中に最初に角を取ったプレイヤーの相手へ、上書き回数+1を付与する（1回のみ）。

### 8.7 ランダム1列カウント除外
- ON時、終局集計でランダムな1行または1列を無視して勝敗を決定する。

## 9. ルーム対戦仕様

### 9.1 役割
- `host` / `guest` / `spectator`
- 非ホストは一部UI（モード・ハンデ・スキル関連設定）の変更不可。

### 9.2 同期イベント
- 着手: `row`, `col`, `useDoubleAction`
- 特殊行動:
  - `immutable-place`
  - `destroy-skill`
- 引き分け投票同期
- ルールモード同期（`local` / `chaos`）

### 9.3 スナップショット同期
- 盤面・手番・終局状態
- ルームルールモード
- ハンデ設定一式
- 上書き残数/上限
- 固定石状態/チャージ
- 駒破壊状態/残回数
- 2回行動ストック/予約状態
- 角連続喪失カウント

## 10. 引き分け投票（合意）

- 対象: `local` と `room`
- 各陣営の投票状態を保持。
- 黒白双方が投票すると即時引き分け終了。

## 11. UI表示仕様

- HUD表示:
  - 手番
  - タイマー
  - モード
  - CPUレベル
  - 先攻後攻
  - 石数（BLACK/WHITE）
- カオス時追加表示:
  - 上書き残り
  - 2回行動ストック
  - スキルパネル（駒破壊 / 固定石指定 / 2回行動発動）
- ルーム時表示:
  - ROOMコード
  - ROLE
  - ロック中オーバーレイ（待機メッセージ）

## 12. 既知の実装メモ

- `REMAKE` は通常の再戦用途に加え、`local/room` では引き分け投票導線としても動作する。
- CPU学習データはユーザー単位キーへ保存されるが、旧キーからの移行処理を持つ。
- カオスは拡張ルール群を含むため、競技オセロの厳密ルールとは一致しない。
